What is ALCHEMY

ALCHEMY는 공개용 SaaS가 아니라 개인 AI 작업 운영체계입니다. 사용자는 세부 실행 계획을 직접 짜지 않습니다. 에이전트가 다음 단계를 권장하고, 사용자는 승인/보류/거절만 선택합니다.

"덜 묻고, 더 판단하고, 꼭 필요한 것만 요청한다."

Decision Philosophy

에이전트가 판단합니다. 사용자는 승인 / 보류 / 거절만 선택합니다.

✅

Approve

에이전트 권장안을 승인하고 즉시 실행

🔒

Hold

추가 검토가 필요할 때 보류

❌

Reject

거절 시 새 권장안 자동 생성

기술적 판단은 에이전트가 맡고,
비용·계정·토큰·외부 연락·개인정보처럼 실제 책임이 생기는 순간에만 사용자에게 묻습니다.

User Approval Boundary

반드시 사용자 승인이 필요한 10가지 상황

1실제 비용 발생
2외부 계정 생성/로그인
3API 키·토큰·비밀키 사용
4실제 고객/외부인 연락
5개인정보/민감정보 사용
6앱스토어/플레이스토어 제출
7결제/구독/유료 상품 공개
8법적 책임/계약 책임 발생
9로컬 설치/시스템 변경
10계정·저장소·배포 영향
